# Fullstack Moonrepo Simple Template

A polyglot monorepo managed by [moonrepo](https://moonrepo.dev), combining a Python (FastAPI + gRPC) backend with a React + Vite frontend.

## Architecture

```
┌─────────────┐   HTTP    ┌──────────────┐   gRPC    ┌─────────────────┐
│   Frontend  │ ───────▶  │   FastAPI    │ ───────▶  │  gRPC UserService │
│ React/Vite  │  :3000    │   Gateway    │  :50051   │   (Python)        │
└─────────────┘           │    :8000     │           └─────────┬─────────┘
                           └──────────────┘                     │
                                                          SQLAlchemy / SQLite
                                                           (in-memory)
```

- **frontend** — React 18 + Vite + TypeScript app that queries a user by email.
- **backend** — A single Python process that starts two servers:
  - A FastAPI HTTP gateway (`:8000`) exposing `GET /api/user?email=...`, which proxies requests over gRPC.
  - A gRPC `UserService` (`:50051`) defined in [`backend/app/proto/user.proto`](backend/app/proto/user.proto) that reads from an in-memory SQLite database via SQLAlchemy.

## Prerequisites

- [moon CLI](https://moonrepo.dev/docs/install)
- Node.js `22.21.1` and [pnpm](https://pnpm.io) (managed automatically by moon's toolchain)
- Python `3.12.13` (managed automatically by moon's toolchain)

Toolchain versions are pinned in [`.moon/toolchain.yml`](.moon/toolchain.yml); moon will provision them for you.

## Getting started

Run both apps in dev mode from the repo root:

```bash
moon run backend:dev
moon run frontend:dev
```

Or run every project's `dev` task in parallel:

```bash
moon run :dev
```

This will:
- Create the backend's Python virtualenv and install dependencies (`backend:setup`)
- Generate gRPC/protobuf Python stubs from `user.proto` (`backend:codegen`)
- Start the FastAPI gateway on `http://localhost:8000` and the gRPC service on port `50051`
- Start the Vite dev server on `http://localhost:3000`

Open `http://localhost:3000` and search for the seeded user (`test@example.com`).

## Project structure

```
backend/
  app/
    main.py           # Boots the FastAPI gateway and gRPC server as separate processes
    api/routes.py      # FastAPI routes that proxy HTTP requests to the gRPC service
    db/database.py      # SQLAlchemy models and in-memory SQLite setup (with seed data)
    proto/user.proto     # gRPC service & message definitions
  requirements.txt
  moon.yml

frontend/
  src/
    App.tsx            # UI for querying a user by email
    main.tsx
  package.json
  vite.config.ts
  moon.yml

.moon/                  # Moonrepo workspace and toolchain configuration
```

## Notes

- The backend uses an **in-memory SQLite database**, so data resets every time the backend restarts. A single user (`id=1`, `test@example.com`) is seeded on startup in [`backend/app/db/database.py`](backend/app/db/database.py).
- Protobuf stubs (`user_pb2.py`, `user_pb2_grpc.py`) are generated by the `backend:codegen` moon task and should not be edited by hand — edit [`user.proto`](backend/app/proto/user.proto) instead and regenerate.
- CORS on the FastAPI gateway is restricted to `http://localhost:3000` (see [`backend/app/main.py`](backend/app/main.py)).

## Available moon tasks

| Project  | Task      | Description                                      |
|----------|-----------|---------------------------------------------------|
| backend  | `setup`   | Creates `.venv` and installs Python dependencies   |
| backend  | `codegen` | Compiles `user.proto` into Python gRPC stubs       |
| backend  | `dev`     | Runs the FastAPI + gRPC servers                    |
| frontend | `dev`     | Runs the Vite dev server                            |
| frontend | `build`   | Type-checks and builds the frontend for production  |
